  9 | # Product Context
 10 | 
 11 | ## What MissionForge Is
 12 | 
 13 | MissionForge is a CLI-first harness for making AI-assisted refactor and modernization work more scoped, measurable, reviewable, and auditable.
 14 | 
 15 | It is designed for IBM Bob-assisted development, but it is not dependent on Bob as a platform. The core idea is that Bob performs the reasoning-heavy work, while MissionForge manages the mission structure and deterministic evidence.
 16 | 
 17 | MissionForge turns a vague task like:
 18 | 
 19 | ```text
 20 | Modernize matchmaking from CORBA to REST without breaking the rest of the game.
 21 | ```
 22 | 
 23 | into a structured mission with:
 24 | 
 25 | - a clear goal
 26 | - forbidden paths
 27 | - allowed paths
 28 | - measurable baseline values
 29 | - target values
 30 | - sub-missions
 31 | - dependency order
 32 | - validation results
 33 | - test evidence
 34 | - a final evidence report
 35 | 
 36 | The key principle is:
 37 | 
 38 | > Bob measures the code. MissionForge measures the mission.
 39 | 
 40 | Bob understands source code. MissionForge does not try to understand source code semantics. MissionForge manages state, scope, validation flow, files, git diff checks, test execution, and reporting.
 41 | 
 42 | ---
 43 | 
 44 | ## What Problem MissionForge Solves
 45 | 
 46 | AI coding agents can refactor quickly, but reviewers often need answers to questions like:
 47 | 
 48 | - What exactly was the agent supposed to change?
 49 | - What files was it not allowed to touch?
 50 | - What was the baseline before the change?
 51 | - What metric proves the refactor worked?
 52 | - Were tests run?
 53 | - Did the agent accidentally modify unrelated code?
 54 | - Can the result be attached to a pull request as evidence?
 55 | - Can the work be broken into smaller reviewable units?
 56 | 
 57 | Without a harness, these answers are usually buried in chat history, terminal logs, or the agent's final summary.
 58 | 
 59 | MissionForge makes these answers explicit through mission files, validation files, and generated reports.
 60 | 
 61 | ---
 62 | 
 63 | ## What MissionForge Is Not
 64 | 
 65 | MissionForge is not a semantic code analyzer.
 66 | 
 67 | The CLI does not:
 68 | 
 69 | - inspect source code meaning
 70 | - resolve symbols
 71 | - understand imports
 72 | - count language-specific references
 73 | - decide whether a refactor is correct
 74 | - replace Bob's reasoning
 75 | 
 76 | Instead, the CLI handles deterministic operations such as:
 77 | 
 78 | - creating mission folders
 79 | - validating YAML and JSON structure
 80 | - tracking mission state
 81 | - checking changed files through git
 82 | - comparing changed files against allowed and forbidden paths
 83 | - running test commands
 84 | - storing baseline and validation records
 85 | - generating Markdown reports
 86 | 
 87 | ---
 88 | 
 89 | ## Roles and Responsibilities
 90 | 
 91 | ## Bob's Role
 92 | 
 93 | Bob is responsible for reasoning-heavy work:
 94 | 
 95 | - reading the codebase
 96 | - understanding the refactor goal
 97 | - decomposing the parent mission into sub-missions
 98 | - defining meaningful metrics
 99 | - filling baseline measurements
100 | - implementing code changes
101 | - filling validation measurements
102 | - explaining the work when needed
103 | 
104 | Bob should use MissionForge as a workflow harness, not bypass it.
105 | 
106 | ## MissionForge CLI's Role
107 | 
108 | The CLI is responsible for deterministic mission management:
109 | 
110 | - initializing mission workspaces
111 | - validating mission and sub-mission schemas
112 | - resolving dependency order
113 | - producing baseline and validation todo files
114 | - running git diff checks
115 | - running user-provided test commands
116 | - committing structured baseline and validation records
117 | - generating evidence reports
118 | 
119 | The CLI is the source of truth for mission state.
120 | 
121 | ## Mission Board's Role
122 | 
123 | The Mission Board is the Phase 2 web UI.
124 | 
125 | It reads MissionForge state and makes it visible through:
126 | 
127 | - Kanban-style mission cards
128 | - parent mission and sub-mission progress
129 | - dependency visualization
130 | - validation status
131 | - evidence report rendering
132 | - stakeholder-friendly summaries
133 | - watsonx.ai mission chat
134 | 
135 | The Board is optional. The CLI must still work without it.
136 | 
137 | ## Watsonx.ai's Role
138 | 
139 | Watsonx.ai is used in Phase 2 for browser-side synthesis over mission state.
140 | 
141 | It can support:
142 | 
143 | - Mission Chat
144 | - Stakeholder Translation
145 | - business-friendly summaries
146 | - risk explanations
147 | - standup summaries
148 | 
149 | Watsonx.ai should answer from MissionForge's structured mission state, not from unrestricted speculation about the source code.

```python
"""Mission validation logic."""

import pathspec
import yaml
from pathlib import Path
from typing import Any

from jsonschema import ValidationError as JSONSchemaValidationError

from .exceptions import ValidationError
from ..schemas import get_mission_schema, get_validator


class MissionValidator:
    """Validates parent mission configuration."""
    
    def __init__(self):
        self.schema = get_mission_schema()
        self.validator = get_validator(self.schema)
    
    def validate_file(self, mission_path: Path) -> tuple[bool, list[str]]:
        """Validate mission.yaml file.
        
        Returns:
            Tuple of (is_valid, error_messages)
        """
        errors = []
        
        # Check file exists
        if not mission_path.exists():
            return False, [f"Mission file not found: {mission_path}"]
        
        # Parse YAML
        try:
            with open(mission_path) as f:
                data = yaml.safe_load(f)
        except yaml.YAMLError as e:
            return False, [f"Invalid YAML syntax: {e}"]
        
        # Validate against schema
        try:
            self.validator.validate(data)
        except JSONSchemaValidationError as e:
            errors.append(f"Schema validation failed: {e.message}")
            return False, errors
        
        # Custom validations
        errors.extend(self._validate_glob_patterns(data.get("forbidden_paths", [])))
        errors.extend(self._validate_aggregate_metrics(data.get("aggregate_metrics", {})))
        errors.extend(self._validate_mission_id_consistency(data, mission_path))
        
        return len(errors) == 0, errors
    
    def _validate_glob_patterns(self, patterns: list[str]) -> list[str]:
        """Validate glob patterns are valid."""
        errors = []
        for pattern in patterns:
            try:
                pathspec.PathSpec.from_lines("gitwildmatch", [pattern])
            except Exception as e:
                errors.append(f"Invalid glob pattern '{pattern}': {e}")
        return errors
    
    def _validate_aggregate_metrics(self, metrics: dict[str, Any]) -> list[str]:
        """Validate aggregate metrics structure."""
        errors = []
        for metric_name, metric_config in metrics.items():
            if not isinstance(metric_config, dict):
                errors.append(f"Metric '{metric_name}' must be an object")
                continue
            
            # Check for at least one constraint (max, min, target)
            if not any(k in metric_config for k in ["max", "min", "target"]):
                errors.append(
                    f"Metric '{metric_name}' must have at least one constraint (max, min, or target)"
                )
        return errors
    
    def _validate_mission_id_consistency(
        self, data: dict[str, Any], mission_path: Path
    ) -> list[str]:
        """Validate mission ID matches directory name."""
        errors = []
        mission_id = data.get("id")
        expected_id = mission_path.parent.name
        
        if mission_id != expected_id:
            errors.append(
                f"Mission ID '{mission_id}' does not match directory name '{expected_id}'"
            )
        return errors
```

- YAML syntax validation

"""Schema definitions and validation utilities."""

import json
from pathlib import Path
from typing import Any

from jsonschema import Draft7Validator

SCHEMA_DIR = Path(__file__).parent


def load_schema(schema_name: str) -> dict[str, Any]:
    """Load JSON schema by name.
    
    Args:
        schema_name: Name of schema file (without .json extension).
        
    Returns:
        Schema dictionary.
        
    Raises:
        FileNotFoundError: If schema file doesn't exist.
    """
    schema_path = SCHEMA_DIR / f"{schema_name}.json"
    with open(schema_path) as f:
        return json.load(f)


def get_mission_schema() -> dict[str, Any]:
    """Get parent mission schema.
    
    Returns:
        Mission schema dictionary.
    """
    return load_schema("mission_schema")


def get_validator(schema: dict[str, Any]) -> Draft7Validator:
    """Get JSON schema validator.
    
    Args:
        schema: JSON schema dictionary.
        
    Returns:
        Configured validator instance.
    """
    return Draft7Validator(schema)

"""Mission validation logic."""

import pathspec
import yaml
from pathlib import Path
from typing import Any

from jsonschema import ValidationError as JSONSchemaValidationError

from .exceptions import ValidationError
from ..schemas import get_mission_schema, get_validator


class MissionValidator:
    """Validates parent mission configuration."""

    def __init__(self) -> None:
        """Initialize validator with mission schema."""
        self.schema = get_mission_schema()
        self.validator = get_validator(self.schema)

    def validate_file(self, mission_path: Path) -> tuple[bool, list[str]]:
        """Validate mission.yaml file.

        Args:
            mission_path: Path to mission.yaml file.

        Returns:
            Tuple of (is_valid, error_messages).
        """
        errors: list[str] = []

        # Check file exists
        if not mission_path.exists():
            return False, [f"Mission file not found: {mission_path}"]

        # Parse YAML
        try:
            with open(mission_path) as f:
                data = yaml.safe_load(f)
        except yaml.YAMLError as e:
            return False, [f"Invalid YAML syntax: {e}"]

        # Validate against schema
        try:
            self.validator.validate(data)
        except JSONSchemaValidationError as e:
            errors.append(f"Schema validation failed: {e.message}")
            return False, errors

        # Custom validations
        errors.extend(self._validate_glob_patterns(data.get("forbidden_paths", [])))
        if "allowed_paths" in data:
            errors.extend(self._validate_glob_patterns(data.get("allowed_paths", [])))
        errors.extend(self._validate_aggregate_metrics(data.get("aggregate_metrics", {})))
        errors.extend(self._validate_mission_id_consistency(data, mission_path))

        return len(errors) == 0, errors

    def _validate_glob_patterns(self, patterns: list[str]) -> list[str]:
        """Validate glob patterns are valid.

        Args:
            patterns: List of glob patterns to validate.

        Returns:
            List of error messages.
        """
        errors: list[str] = []
        for pattern in patterns:
            try:
                pathspec.PathSpec.from_lines("gitwildmatch", [pattern])
            except Exception as e:
                errors.append(f"Invalid glob pattern '{pattern}': {e}")
        return errors

    def _validate_aggregate_metrics(self, metrics: dict[str, Any]) -> list[str]:
        """Validate aggregate metrics structure.

        Args:
            metrics: Metrics dictionary to validate.

        Returns:
            List of error messages.
        """
        errors: list[str] = []
        for metric_name, metric_config in metrics.items():
            if not isinstance(metric_config, dict):
                errors.append(f"Metric '{metric_name}' must be an object")
                continue

            # Check for at least one constraint (max, min, target)
            if not any(k in metric_config for k in ["max", "min", "target"]):
                errors.append(
                    f"Metric '{metric_name}' must have at least one constraint (max, min, or target)"
                )
        return errors

    def _validate_mission_id_consistency(
        self, data: dict[str, Any], mission_path: Path
    ) -> list[str]:
        """Validate mission ID matches directory name.

        Args:
            data: Mission data dictionary.
            mission_path: Path to mission.yaml file.

        Returns:
            List of error messages.
        """
        errors: list[str] = []
        mission_id = data.get("id")
        expected_id = mission_path.parent.name

        if mission_id != expected_id:
            errors.append(
                f"Mission ID '{mission_id}' does not match directory name '{expected_id}'"
            )
        return errors

from ...core.mission_validator import MissionValidator
from ...core.workspace import Workspace

console = Console()


@app.command("validate")
def validate_mission(
    mission_id: str = typer.Argument(..., help="Mission ID to validate"),
) -> None:
    """Validate parent mission configuration."""

    try:
        workspace = Workspace()
    except Exception as e:
        console.print(f"[red]Error:[/red] {e}")
        raise typer.Exit(1)

    mission_path = workspace.mission_path(mission_id)
    mission_file = mission_path / "mission.yaml"

    if not mission_file.exists():
        console.print(f"[red]Error:[/red] Mission {mission_id} not found")
        console.print(f"Expected: {mission_file}")
        raise typer.Exit(1)

    # Validate mission
    validator = MissionValidator()
    is_valid, errors = validator.validate_file(mission_file)

    if is_valid:
        console.print(
            Panel(
                f"[green]✓[/green] Mission {mission_id} is valid",
                title="Validation Success",
                border_style="green",
            )
        )
        console.print("\n[bold]Next steps:[/bold]")
        console.print(f"1. Run: missionforge mission decompose {mission_id}")
        console.print("2. Or manually edit sub-missions in mission.yaml")
    else:
        console.print(
            Panel(
                "[red]✗[/red] Mission validation failed",
                title="Validation Errors",
                border_style="red",
            )
        )

        table = Table(show_header=False, box=None)
        table.add_column("Error", style="red")

        for error in errors:
            table.add_row(f"• {error}")

        console.print(table)
        console.print(f"\n[yellow]Fix errors in:[/yellow] {mission_file}")
        raise typer.Exit(1)

```bash
# Clone the repository
```

```bash
# Create a new mission workspace
missionforge init MF-001

missionforge workspace init MF-001
```

```
.missionforge/
```

```bash
# Validate mission.yaml structure and constraints
missionforge mission validate MF-001
```
